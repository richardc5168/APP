# Parent Report V2 Spec

目的：把家長報告從「資料展示」提升成「願意續訂的價值展示」。

## 1. 頁面位置

- 頁面：`docs/parent-report/index.html`
- 目標受眾：家長
- 主要價值：看懂孩子弱點、立即知道下一步要練什麼

## 2. 目前已實作區塊

### 本週表現摘要

- 最近 24 小時快照
- 最近 7 天總覽
- 作答量、正確率、提示依賴、每日作答量
- 新增「本週重點摘要」：完成題數、正確率、主動完成天數、提示使用比例

### 概念掌握雷達

- 分數
- 小數
- 百分率
- 體積
- 生活應用

備註：需求原始版本寫「應用題」，目前實作以生活應用為對應維度。

### 問題診斷

- 錯題摘要
- 詳細錯題分析
- 依錯誤模式推論卡點原因

### 下週補救建議

- 補救優先順序列表
- 推薦題組深連結
- 家長建議文案
- 新增「本週 AI 補救建議」：只列 3 件最值得先做的具體動作

## 3. 付費 gating

- 基礎資料可看
- 完整雷達與完整補救建議需付費
- 實作依據：`AIMathSubscription.canAccessFullReport()`

## 4. UI 原則

- 不說抽象鼓勵，直接指出弱點
- 每個建議都可執行
- 報表像付費功能，而不是附屬頁面

## 5. 資料聚合來源

- `attempt_telemetry.js`
- `coach_event_log.js`
- 雲端同步學生資料

## 6. 應補齊但尚未完全獨立成區塊的需求

以下需求已部分具備基礎，但仍可再收斂成明確區塊：

- 本週 AI 補救建議
- 建議先練哪三題型
- 比上週進步 / 退步

目前狀態：

- 補救建議已有，且已收斂為 3 個最優先動作
- 題型推薦已有深連結
- 與上週比較已有獨立週對比卡片

## 7. Acceptance Criteria

- 家長打開頁面 30 秒內看懂弱點
- 首屏就能看到「這週先看哪四個指標」與「先做哪三件事」
- 建議可以直接點進推薦題組
- 免費與付費差異清楚
- 週報具備續訂理由，而不是只列數據
