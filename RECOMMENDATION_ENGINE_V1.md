# Recommendation Engine V1

目的：用最小規則式方法，將錯題與提示依賴轉成可執行的補強順序。

## 1. 實作位置

- 主要頁面：`docs/parent-report/index.html`
- 輸入資料：attempt telemetry、錯題摘要、模組與 topic 分布

## 2. 核心規則

### 規則 1：錯題量高優先

- 某模組錯題數越高，優先級越高

### 規則 2：提示依賴高優先

- `max_hint` 越高，代表概念穩定度越低

### 規則 3：生活應用與分數文字題優先拉高

- 若同時有多個弱點，文字理解與分量應用優先

### 規則 4：推薦必須可點

- 每個建議都附模組連結
- 不給抽象語句，不只說「多練習」

## 3. 推薦輸出格式

建議每筆推薦至少包含：

```json
{
  "priority": 1,
  "concept": "分數應用",
  "reason": "錯題多且提示依賴高",
  "module": "fraction-word-g5",
  "link": "../fraction-word-g5/"
}
```

## 4. 目前推薦面呈現

- 補救優先順序
- 推薦模組深連結
- 家長建議文案

## 5. 後續升級方向

- 加入「比上週進步/退步」權重
- 加入主題完成率權重
- 加入週報點擊後回流率關聯
